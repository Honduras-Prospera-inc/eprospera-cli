# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Status

`eprospera-cli` (`@prospera/eprospera-cli`) is a TypeScript CLI over the e-Prospera public API, currently in v0.1 scaffold state. The tooling, config, design docs, and `cli.ocs.yaml` are in place, but **most `src/**/*.ts` files are 0-byte placeholders waiting to be implemented**. The exception is `src/api/generated.ts`, which is generated by `pnpm run gen:api` from the upstream OpenAPI spec — never hand-edit it.

The authoritative design lives in `docs/scaffold/` (numbered 01-10). Read those before changing architectural decisions; the build order is `docs/scaffold/07-build-plan.md`. Tracked generated command docs live in `docs/commands/` and should be regenerated from `cli.ocs.yaml`.

## Common Commands

Package manager is **pnpm** (pinned to `pnpm@11.3.0`). Node >= 20.

- `pnpm build` — TypeScript build to `dist/`
- `pnpm test` — Vitest run (matches `test/**/*.test.ts`)
- `pnpm test <pattern>` — run a single test file or filter (e.g. `pnpm test ocs`)
- `pnpm typecheck` — `tsc --noEmit`
- `pnpm check` — Biome lint/format + typecheck (use this before commits)
- `pnpm validate:ocs` — validate `cli.ocs.yaml` against the OpenCLI JSON schema
- `pnpm run gen:all` — regenerate API types, docs, and shell completions (runs as a precommit hook via `simple-git-hooks`)
- `pnpm run gen:api` — regenerate `src/api/generated.ts` from `EPROSPERA_OPENAPI_URL` (defaults to `https://docs.eprospera.com/openapi.yaml`)
- `pnpm run gen:docs` / `gen:completions` — regenerate `docs/commands/` and `dist/completions/`

E2E tests against staging are opt-in: set `EPROSPERA_E2E=1` and `EPROSPERA_API_KEY=<fixture-ak>` (see `docs/scaffold/08-testing-and-release.md`).

The binary entry is `bin/eprospera.js`, which simply imports `dist/src/index.js` — so the CLI must be built before it can be run from disk.

## Architecture

### `cli.ocs.yaml` is the source of truth

The command surface, global flags, exit codes, and help-text examples all live in `cli.ocs.yaml` (OpenCLI Specification). The Commander wiring in `src/index.ts`, the README usage block, the per-command docs in `docs/commands/`, and the shell completions in `dist/completions/` are all generated from or validated against it. **Hand-edited drift in generated surfaces is a CI failure** — always update `cli.ocs.yaml` first, then run `pnpm run gen:all`.

The `eprospera schema` subcommand prints `cli.ocs.yaml` verbatim; agents are expected to ingest it to discover the command tree.

### Layered runtime (target structure, see `docs/scaffold/06-repository-layout.md`)

- `src/api/` — typed `openapi-fetch` client (`client.ts`), generated types (`generated.ts`), and API → CLI error mapping (`errors.ts`). Base URL resolved from `EPROSPERA_BASE_URL` → `EPROSPERA_ENV=staging` → production default. Adds bearer auth, exponential-backoff retry on 5xx/429 (cap 3, honor `Retry-After`), and a UUIDv4 `Idempotency-Key` for writes.
- `src/credentials/` — resolution precedence is `--api-key` flag → `EPROSPERA_API_KEY` env → `keytar` keychain → `$XDG_CONFIG_HOME/eprospera-cli/credentials.json` (mode `0600`). Stored credentials carry `kind: "ak" | "sk" | "oauth"` plus scopes/expiry.
- `src/scopes/` — static command-to-scope map (`map.ts`) and local preflight (`check.ts`). Preflight runs before every API call; missing scope exits `4` without making the network request. `--skip-scope-check` opts out.
- `src/output/` — three output modes: `human` (default, TTY), `json` (`--json` or non-TTY unless `--no-auto-json`), `raw` (`--raw`, compact). **JSON goes to stdout; everything else (logs, spinners, prompts, errors in human mode) goes to stderr.** Stdout must remain parseable JSON in machine modes, even on error. `--fields a,b,c` masks output keys.
- `src/polling/watch.ts` — generic poll loop. `application watch` polls every 30s for the first 10 minutes, then every 5 minutes; default 30-minute timeout. In JSON mode it emits NDJSON, one object per state transition.
- `src/prompts/confirm.ts` — destructive writes (`application create`, `application pay`) prompt in interactive TTY mode; suppressed by `--yes`/`-y` or non-TTY.
- `src/errors.ts` — `ExitError` with code + machine envelope `{ error: { code, message, httpStatus, details } }`.

### Exit codes (stable contract; agents branch on them)

`0` ok · `1` generic · `2` bad usage · `3` auth (no/expired credential) · `4` authorization (missing scope) · `5` not found · `6` conflict · `7` rate-limited (surface `Retry-After`) · `8` validation (surface `details`) · `9` timeout · `10` terminal failure state (e.g. `Rejected`, `PaymentFailed`).

### Each command handler should

1. Define a Zod schema for inputs.
2. Run the scope check.
3. Call the typed `openapi-fetch` client.
4. Route the result through `src/output/format.ts` (single `print(data, { schema })` entry point).
5. Exit with the right code per the table above.

## Conventions

- **TypeScript strict mode, ESM only** (`"type": "module"`, NodeNext). The tsconfig sets `noUncheckedIndexedAccess`, `noUnusedLocals`, `noUnusedParameters`, `noImplicitOverride`, `noFallthroughCasesInSwitch` — write to that bar.
- **Biome** for lint/format: 2-space indent, 100-col line width, double quotes, semicolons, trailing commas. `pnpm check` runs `biome check` then `tsc --noEmit`.
- **Validation at the CLI boundary with `zod`** — mirrors upstream API validation; do not bypass.
- **No telemetry.** This CLI handles bearer tokens; do not phone home, do not log tokens.
- **Don't re-litigate fixed decisions** (the table in `docs/scaffold/02-decisions-and-constraints.md`). Anything outside that table needs a new ADR in `docs/decisions/` (existing: commander-over-oclif, pnpm-workspace, simple-git-hooks).
- **Changesets** manages releases. Add a changeset with `pnpm exec changeset` when changing published behavior; `commit: false` means changesets won't auto-commit for you.
